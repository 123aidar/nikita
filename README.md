# Продуктовый магазин - Система управления

Современная веб-система для управления продуктовым магазином, разработанная на Django с красивым и интуитивным интерфейсом.

## 🚀 Возможности

### 📦 Управление товарами
- Каталог товаров с изображениями и категориями
- Отслеживание количества на складе
- Контроль сроков годности
- История изменения цен

### 🚚 Управление поставками
- Регистрация поставок товаров
- Автоматическое обновление остатков
- Учет поставщиков

### 💰 Регистрация продаж
- Быстрая регистрация продаж
- Автоматический расчет суммы
- История всех продаж

### 📊 Инвентаризация
- Списание товаров (просрочка, брак)
- Проведение инвентаризаций
- Учет расхождений

### 📈 Отчеты и аналитика
- Графики прибыли (неделя, месяц, год)
- Топ продаваемых товаров
- Статистика по продажам
- Интерактивные графики (Chart.js)

### 💬 Чат для сотрудников
- Общение в реальном времени (WebSocket)
- Поддержка эмодзи
- Аватары пользователей
- Красивый мобильный интерфейс

### 👥 Управление пользователями
- Две роли: Заведующий, Кассир
- Разграничение прав доступа
- Личные профили с аватарами

## 🎨 Дизайн

- Современный градиентный дизайн
- Плавные анимации (Animate.css)
- Адаптивный интерфейс (Bootstrap 5)
- Иконки Font Awesome
- Интерактивные элементы

## 📋 Установка и запуск

### 1. Клонирование и установка зависимостей

```bash
# Создайте виртуальное окружение
python -m venv venv

# Активируйте виртуальное окружение
# На Windows:
venv\Scripts\activate
# На Linux/Mac:
source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt
```

### 2. Настройка базы данных

```bash
# Создайте миграции
python manage.py makemigrations

# Примените миграции
python manage.py migrate
```

### 3. Создание суперпользователя

```bash
python manage.py createsuperuser
```

Следуйте инструкциям для создания учетной записи администратора.

### 4. Создание тестовых данных (опционально)

Откройте Django shell:

```bash
python manage.py shell
```

Выполните следующий код:

```python
from accounts.models import CustomUser
from products.models import Category, Product

# Создание категорий
categories = [
    {'name': 'Овощи', 'icon': 'fa-carrot'},
    {'name': 'Фрукты', 'icon': 'fa-apple-alt'},
    {'name': 'Молочные продукты', 'icon': 'fa-cheese'},
    {'name': 'Мясо', 'icon': 'fa-drumstick-bite'},
    {'name': 'Хлеб', 'icon': 'fa-bread-slice'},
    {'name': 'Напитки', 'icon': 'fa-wine-bottle'},
]

for cat_data in categories:
    Category.objects.get_or_create(**cat_data)

# Создание пользователей с разными ролями
users = [
    {'username': 'manager', 'password': 'manager123', 'first_name': 'Мария', 'last_name': 'Сидорова', 'role': 'manager', 'email': 'manager@shop.ru'},
    {'username': 'cashier', 'password': 'cashier123', 'first_name': 'Анна', 'last_name': 'Иванова', 'role': 'cashier', 'email': 'cashier@shop.ru'},
]

for user_data in users:
    password = user_data.pop('password')
    user, created = CustomUser.objects.get_or_create(
        username=user_data['username'],
        defaults=user_data
    )
    if created:
        user.set_password(password)
        user.save()
        print(f"Создан пользователь: {user.username} (пароль: {password})")

print("Тестовые данные созданы!")
```

### 5. Запуск сервера

```bash
# Для работы чата используйте Daphne (ASGI сервер)
python manage.py runserver
# или
daphne -b 0.0.0.0 -p 8000 grocery_store.asgi:application
```

Откройте браузер и перейдите по адресу: http://localhost:8000

## 📸 Добавление изображений

Для добавления изображений товаров:

1. Войдите в административную панель: http://localhost:8000/admin
2. Перейдите в раздел "Товары"
3. Создайте или отредактируйте товар
4. Загрузите изображение в поле "Изображение"

Для добавления аватаров пользователей:
1. Войдите в систему
2. Перейдите в "Профиль" в меню пользователя
3. Загрузите изображение аватара

## 📝 Дополнительная настройка

### Изменение секретного ключа (для продакшн)

В файле `grocery_store/settings.py` замените:
```python
SECRET_KEY = 'django-insecure-your-secret-key-here-change-in-production'
```

### Настройка базы данных PostgreSQL

В файле `grocery_store/settings.py` измените:
```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'grocery_store',
        'USER': 'your_username',
        'PASSWORD': 'your_password',
        'HOST': 'localhost',
        'PORT': '5432',
    }
}
```

### Настройка Redis для WebSocket (продакшн)

```python
CHANNEL_LAYERS = {
    'default': {
        'BACKEND': 'channels_redis.core.RedisChannelLayer',
        'CONFIG': {
            "hosts": [('127.0.0.1', 6379)],
        },
    },
}
```

## 🐛 Решение проблем

### WebSocket не работает
- Убедитесь, что используете Daphne вместо стандартного runserver
- Проверьте настройки ASGI в `asgi.py`
- Для продакшн настройте Redis

### Изображения не отображаются
- Убедитесь, что настроены MEDIA_URL и MEDIA_ROOT
- Проверьте, что в `urls.py` добавлены маршруты для медиа-файлов
- В продакшн настройте обслуживание медиа через Nginx
