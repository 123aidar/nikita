# 🔧 РЕШЕНИЕ ПРОБЛЕМЫ С ПОРТОМ 8000

## ❌ Проблема:
```
Listen failure: Couldn't listen on 127.0.0.1:8000
[WinError 10013] Сделана попытка доступа к сокету методом, запрещенным правами доступа
```

Это означает, что порт 8000 уже занят другим процессом (у вас это "Manager" PID 4940).

---

## ✅ РЕШЕНИЕ 1: Автоматическое (РЕКОМЕНДУЕТСЯ)

Просто запустите **`start_server.bat`** заново!

Теперь скрипт автоматически:
1. Проверит, занят ли порт 8000
2. Предложит освободить порт или использовать порт 8080
3. Запустит сервер

---

## ✅ РЕШЕНИЕ 2: Использовать другой порт (БЫСТРО)

Запустите: **`start_server_alt_port.bat`**

Сервер запустится на порту **8080**: http://127.0.0.1:8080/

---

## ✅ РЕШЕНИЕ 3: Завершить процесс вручную

### Шаг 1: Запустите от имени администратора
**ПКМ на `kill_port_8000.bat` → Запуск от имени администратора**

### Шаг 2: Запустите сервер
**`start_server.bat`**

---

## ✅ РЕШЕНИЕ 4: Через PowerShell с правами админа

1. Запустите PowerShell от имени администратора
2. Выполните:
```powershell
cd d:\1
Get-NetTCPConnection -LocalPort 8000 | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
python manage.py runserver
```

---

## ✅ РЕШЕНИЕ 5: Полный сброс с альтернативным портом

Запустите: **`full_reset_and_start.bat`**

(Теперь использует порт 8080)

---

## 🎯 РЕКОМЕНДАЦИЯ:

**Самый простой способ:**
1. Закройте все окна Python/Django
2. Запустите **`start_server.bat`**
3. Выберите пункт [2] - запуск на порту 8080
4. Откройте: **http://127.0.0.1:8080/**

---

## 📝 ПРИМЕЧАНИЕ:

Если вы запускаете на порту 8080, в браузере используйте:
- ❌ НЕ: http://127.0.0.1:8000/
- ✅ ДА: http://127.0.0.1:8080/

Чат будет работать на любом порту!
