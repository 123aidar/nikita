# 🚀 БЫСТРЫЙ СТАРТ

## Шаг 1: Установка зависимостей

Дважды кликните на файл `start.bat` (Windows) или выполните:

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

## Шаг 2: Создание базы данных

```bash
python manage.py makemigrations
python manage.py migrate
```

## Шаг 3: Создание суперпользователя

```bash
python manage.py createsuperuser
```

## Шаг 4: Создание тестовых данных (опционально)

Дважды кликните на `setup_test_data.bat` или выполните:

```bash
python manage.py shell < create_test_data.py
```

Это создаст:
- 2 пользователя (заведующий, кассир)
- 8 категорий товаров
- 23 товара с ценами и остатками

## Шаг 5: Запуск сервера

```bash
python manage.py runserver
```

Откройте браузер: http://localhost:8000

## 📸 Добавление изображений

### Изображения товаров:
1. Перейдите в админ-панель: http://localhost:8000/admin
2. Войдите под суперпользователем
3. Выберите "Товары" → выберите товар
4. Загрузите изображение

### Аватары пользователей:
1. Войдите в систему
2. Нажмите на имя пользователя → "Профиль"
3. Загрузите аватар

## 🔧 Решение проблем

### Ошибка миграций
```bash
python manage.py makemigrations
python manage.py migrate --run-syncdb
```

### Чат не работает
Чат использует WebSocket. В режиме разработки работает через Django Channels с In-Memory backend. Для продакшн нужен Redis.

### Изображения не отображаются
Убедитесь что папки `media/` и `static/` созданы.

## 📞 Полезные команды

```bash
# Создать суперпользователя
python manage.py createsuperuser

# Запустить сервер
python manage.py runserver

# Запустить с ASGI (для WebSocket)
daphne -b 0.0.0.0 -p 8000 grocery_store.asgi:application

# Открыть Django shell
python manage.py shell

# Создать резервную копию БД
python manage.py dumpdata > backup.json

# Восстановить из резервной копии
python manage.py loaddata backup.json
```
